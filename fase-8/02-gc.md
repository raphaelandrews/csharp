# 02 — Garbage Collector do .NET

## O que você realmente precisa saber

O GC do .NET é geracional, com 3 gerações (0, 1, 2). Objetos nascem na Gen0, sobrevivem a uma coleta e vão para Gen1, sobrevivem de novo e vão para Gen2. Objetos grandes (>85KB) vão direto para LOH (Large Object Heap).

Isso importa porque:
- **Gen0**: coleta rápida (~1ms), acontece frequentemente. Alocar e descartar objetos de curta duração é barato.
- **Gen2**: coleta lenta (~10-50ms), rara, pausa o mundo. Objetos que chegam na Gen2 e ficam vivos custam caro.
- **LOH**: não é compactado por padrão (pode fragmentar). Arrays grandes, strings longas.

## Alocação: o que evitar

### 1. Boxing de value types

```csharp
// RUIM: boxing implícito — int vira object, alocado no heap
int count = 42;
logger.LogInformation("Count: {Count}", count); // boxing

// BOM: template string — sem boxing (Serilog e logger source-gen)
logger.LogInformation("Count: {Count}", count); // Serilog trata sem boxing

// Ideal: compile-time logging source generator (.NET 6+)
[LoggerMessage(Level = LogLevel.Information, Message = "Count: {Count}")]
static partial void LogCount(ILogger logger, int count);
```

### 2. Alocação em loops quentes

```csharp
// RUIM: aloca nova string a cada iteração
for (int i = 0; i < 10000; i++)
{
    var result = $"{prefix}_{i}_{suffix}"; // alocação
}

// BOM: Span<T> + stackalloc (zero alocação no heap)
Span<char> buffer = stackalloc char[128];
for (int i = 0; i < 10000; i++)
{
    prefix.TryCopyTo(buffer);
    i.TryFormat(buffer[prefix.Length..], out _);
    suffix.TryCopyTo(buffer[(prefix.Length + intFormatLength)..]);
}
```

### 3. LINQ materializando coleções intermediárias

```csharp
// RUIM: ToList() materializa resultado intermediário
var result = habits
    .Where(h => h.IsActive)      // IEnumerable (lazy)
    .ToList()                     // materializa — alocação
    .Where(h => h.WeeklyFrequency > 3)
    .ToList();                    // materializa de novo

// BOM: compõe antes de materializar
var result = habits
    .Where(h => h.IsActive && h.WeeklyFrequency > 3)
    .ToList();                    // materializa uma vez
```

### 4. Concatenar strings em loop (usar StringBuilder)

```csharp
// RUIM: cada concatenação aloca nova string
string report = "";
foreach (var h in habits)
    report += $"{h.Name}\n";    // N alocações!

// BOM: StringBuilder — aloca uma vez
var sb = new StringBuilder();
foreach (var h in habits)
    sb.AppendLine(h.Name);
string report = sb.ToString();
```

## Modos de GC

| Modo | Comportamento | Quando usar |
|---|---|---|
| **Workstation** (padrão) | Coleta concorrente, pausas curtas | APIs, desktop, dev |
| **Server** | GC por CPU core, pausas mais longas, throughput maior | Produção, containers multi-core |

```xml
<!-- .csproj -->
<ServerGarbageCollection>true</ServerGarbageCollection>
```

Ou via runtime config:
```json
// runtimeconfig.json
{ "configProperties": { "System.GC.Server": true } }
```

## ArrayPool<T>: reuso de buffers

```csharp
// RUIM: aloca array novo a cada requisição
byte[] buffer = new byte[4096];
await stream.ReadAsync(buffer);

// BOM: aluga do pool, devolve depois
byte[] buffer = ArrayPool<byte>.Shared.Rent(4096);
try
{
    await stream.ReadAsync(buffer);
}
finally
{
    ArrayPool<byte>.Shared.Return(buffer);
}
```

Para cenários de alta alocação (leitura de streams, serialização), `ArrayPool<T>` reduz a pressão do GC em 80-90%.
